💡 開發流程建議

建議學員採用以下開發流程，充分運用 Bob 的 Plan 與 Code 模式：

1️⃣ 規劃階段 (Plan 模式)

  • 使用 Plan 模式分析需求與設計架構
  • 讓 Bob 協助規劃 Entity、Repository、Service、Controller 的設計
  • 確認資料庫結構與 API 端點設計
  • 產出開發計畫與實作步驟
  • 可請 Bob 產出系統架構圖、資料流程圖、ER 圖等視覺化文件

2️⃣ 開發階段 (Code 模式)

  • 切換至 Code 模式進行實際開發
  • 依照規劃逐步實作各層級元件
  • 使用 Bob 的程式碼生成與重構功能
  • 執行測試並驗證功能

3️⃣ 模式切換技巧

  • 遇到設計問題時，切回 Plan 模式重新規劃
  • 需要程式碼實作時，使用 Code 模式
  • 善用兩種模式的優勢，提升開發效率

💡 提示：Plan 模式適合思考與設計，Code 模式適合實作與除錯。靈活切換可以讓開發更有條理！

🎯 挑戰概述

組別任務

你/妳們是開發工程師，公司需要為現有的交通卡交易系統新增旅程使用分析功能，以了解乘客行為、優化班次安排等其他功能，並支援營運決策。

你們將使用 IBM Bob 作為 AI 開發助手，從現有的 Sample Code 基礎上，共同快速實作這個數據導向的分析功能。

⚠️ 重要提醒

這是一個開放式挑戰，我們不會提供詳細的實作步驟。你需要：

  • 自行分析需求並設計解決方案
  • 善用 IBM Bob 的各種功能來加速開發

🎓 學習目標

完成本挑戰後，你/妳將能夠：

  • 使用 Bob 從現有程式碼快速新增功能
  • 建立自訂 Slash Command 提升開發效率
  • 撰寫專案 Rules 確保程式碼品質
  • 建立 AGENTS.md 指導 AI 助手
  • 實作複雜的資料聚合與統計分析

📋 功能需求

各分組實作以下核心功能：

📊 模組 1：熱門路線分析

功能說明：統計最多人搭乘的起迄站組合 Top 10

  • ✅ 統計各起迄站組合的搭乘次數
  • ✅ 計算各路線的總收入
  • ✅ 依搭乘次數排序，取前 10 名

要實作的功能：

  • 建立 Journey Entity（旅程記錄）
  • 建立 JourneyRepository 與查詢方法
  • 建立 JourneyService 實作分析邏輯
  • 建立 JourneyController 提供 API
  • API 端點：GET /api/journeys/popular-routes

⏰ 模組 2：尖峰時段分析

功能說明：統計各時段（0-23 小時）的進站人數

  • ✅ 依小時（0-23）統計進站人數
  • ✅ 識別尖峰時段（人數最多的時段）
  • ✅ 回傳完整的 24 小時分布資料

要實作的功能：

  • 在 JourneyRepository 新增時段查詢方法
  • 在 JourneyService 實作時段分析邏輯
  • 在 JourneyController 新增 API 端點
  • API 端點：GET /api/journeys/peak-hours

✅ 現有系統架構

Sample Code 已提供以下基礎功能：

元件 說明
Card Entity 一卡通資料（卡號、餘額、類型）
Station Entity 車站資料（站名、路線）
Transaction Entity 交易記錄（進站、出站記錄）
Repository 層 基本的資料存取介面
H2 資料庫 記憶體資料庫，含測試資料

🔌 待實作功能

你需要實作以下核心元件：

元件 說明
Journey Entity 旅程記錄（起迄站、時間、票價、時長）
JourneyRepository 旅程資料存取介面（含複雜查詢）
JourneyService 旅程分析業務邏輯
JourneyController RESTful API 端點
DTO 類別 API 回應資料結構

💡 實作建議

建議實作順序

  1. 分析現有系統
    • 檢視現有的 Card、Station、Transaction Entity
    • 了解資料庫結構與關聯
    • 查看 data.sql 中的測試資料
  2. 設計 Journey Entity
    • 定義旅程記錄所需的欄位
    • 建立與 Station 的關聯
    • 考慮如何儲存時間與金額資料
  3. 建立 Repository 層
    • 建立 JourneyRepository 介面
    • 設計查詢方法來支援兩個分析功能
    • 考慮使用聚合查詢（分組、計數、排序）
  4. 實作 Service 層
    • 建立 JourneyService 類別
    • 實作熱門路線分析邏輯
    • 實作尖峰時段分析邏輯
    • 處理資料轉換與業務規則
  5. 建立 Controller 層
    • 建立 JourneyController 類別
    • 實作兩個 API 端點
    • 定義適當的回應格式
  6. 測試與驗證
    • 撰寫單元測試
    • 使用 Postman 或瀏覽器測試 API
    • 驗證資料正確性

技術考量重點：

  • 如何有效地進行資料分組與統計？
  • 如何從時間欄位提取小時資訊？
  • 如何設計清晰的 API 回應結構？
  • 如何確保查詢效能？

🤖 產出要求

除了功能實作，你還需要建立以下 Bob 相關文件：

必要產出

  • 自訂 Slash Command：建立 /security-scan 指令
    用途：讓 Bob 執行程式碼安全掃描，檢查 SQL Injection、輸入驗證、敏感資料處理等安全問題
  • 編碼規範：撰寫 .bob/rules/coding-standards.md
  • 安全規範：撰寫 .bob/rules/security-rules.md
  • AI 指引：建立 .bob/AGENTS.md

    • 主題：捷運系統開發最佳實踐（資料驗證、查詢優化、安全處理）

💡 提示

這些文件是幫助你更有效使用 Bob 的關鍵工具。Slash Command 讓 Bob 執行特定的程式碼安全檢查。

🚀 啟動應用程式

按照以下步驟啟動 Spring Boot 應用程式：

步驟 1：進入專案目錄

cd topics/journey-analysis

步驟 2：啟動應用程式

選擇以下任一方式：

方式 A：使用 Maven Wrapper（推薦）

./mvnw spring-boot:run

方式 B：使用已安裝的 Maven

mvn spring-boot:run

步驟 3：驗證啟動成功

看到以下訊息表示啟動成功：

Started JourneyAnalysisApplication in X.XXX seconds

步驟 4：存取應用程式

  • Web UI: http://localhost:8080
  • H2 資料庫控制台: http://localhost:8080/h2-console
    • JDBC URL: jdbc:h2:mem:metro_db
    • Username: sa
    • Password: (留空)
  • API 文件: http://localhost:8080/swagger-ui.html

⚠️ 常見問題

Q: Port 8080 已被佔用？

A: 修改 src/main/resources/application.yml 中的 server.port

Q: Maven 下載依賴很慢？

A: 設定 Maven 使用國內鏡像源

Q: 如何停止應用程式？

A: 在終端機按 Ctrl + C
